Use case description

Use case table

Use case name Description Actors Pre-conditions Post-conditions Steps
Login A user login to System to access the functionality of the system Non-logged user System must be connected to the network. A user must have an account already. After a successfull login the user is sent to the main page Enter username/password -> Validate username/password -> Return user role -> Allow access to the System
Register A user creates an account to be able to access the System Non-logged user The System must be connected to the network.The user must enter a username,password,real name,email,affiliation After a successfull registration a mail is sent to the user to confirm the account The user must complete the registration form
Open conference's site A user can open a conference site to be able to see the details about the conference. Logged user A user must be logged in to be able to access the conference site The user will be sent to the conference site User opens conference list -> Select the conference the user wants to see
Attend a conference section A user can attend any conference he the user wishes from the list of conferences. Logged user A user must be logged in to be able to attend a conference.The user must be the fee to attend the conference. A message will be sent to the user after he pays the fee to inform him that he can attend the conference The user must make a request to attend -> he must pay the fee -> a message that his request was approved is sent
Create a conference A user can create a new conference and also can invite reviewers and also a Co-Chair Chair The user has to be logged in and he must have the status of Chair A new conference is created and an email is sent by the Chair of the conference to other users to invite them as a Co-Chair or as reviewers Creates a new conference -> invites a Co-Chair and reviewers
Change conferences deadlines The Chair of a conference can change the deadlines whenever he wishes Chair The user has to be logged in as a Chair An email is sent to every user who will attend the conference that the deadline was changed Go to conference deadlines -> change the deadlines -> Send the email with the new deadline
Assign a paper for review After a paper is submitted by an Author a Co-Chair has the responsability to assign reviewwers to it Co-Chair The user has to be logged in as a Chair or as a Co-Chair. Also the paper must be already submitted. An email is sent to the reviewers who were assigned a papee Author submits paper -> A Co-Chair assigns reviewers for the paper -> Send an email to the reviewers
Review undecided paper A Co-Chair must review a paper in case no review was done for the paper or if the evaluation of the paper is incomplete Co-Chair The user has to be logged in as a Chair or Co-Chair An email is send with the review to the Author after the review is done
Create a section Different sections for the conferences are created. For every section create a Co-Chair must choose a session chair and also the accepted papers for the conference Co-Chair The user must be logged in a Chair or Co-Chair. The papers submitted by Authors had to be reviewed already An email is sent to the user selected as session chair Create conference section -> choose session chair -> choose accepted papers -> Send notification emails
Submit paper A paper is submitted by an Author to be reviewed Author The user has to be an Author to submit a paper. Also the digital format for the paper has to be a pdf file otherwise the system will not accept it The paper is submitted for review. A message will appear if the paper was submitted without any problem Submit paper -> Verify the format -> Send message if the paper was submitted successfully
See his reviews After the paper is reviewed the user can see all the reviews of his paper Author The user has to be the author of the paper to see the reviews The user has access to all reviews about his paper User opens reviews list -> Select from the list a review for the paper
Bid abstract A reviewer can mention what kind of paper he would like to review Reviewer A user has to be logged in as a reviewer The user will mostly receive the kind of apper he wants to review
Review papers A user who has the rank of Reviewer can review other people papers and to evaluate them giving each aper a score as well Reviewer A user have to be logged in as a reviewer to be able to review papers An email with review is send to the author of the paper
See reviews of reviewed papers A user who has the rank of reviewer can see any review of any paper Reviewer A user has to be logged in as a reviewer to be able to see reviews of reviewed papers User opens reviews list -> Select the review for the paper he is interested in
